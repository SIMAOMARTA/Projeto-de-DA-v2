Detailed Description

Configuração parseada a partir do ficheiro de registos.

Armazena o número de registos físicos disponíveis e a variante do algoritmo a utilizar na alocação de registos.

Variantes suportadas:

  • "basic" : coloração greedy sem fallback (T2.1). Tenta colorir o grafo com N cores; se não for possível, faz spill ilimitado.
  • "spilling" : basic + até K spills de webs (T2.2). Itera de 0 a K spills parando assim que a alocação for viável.
  • "splitting" : basic + até K splits de webs (T2.3). Divide webs com grau elevado antes de tentar a coloração.
  • "free" : algoritmo próprio baseado na heurística de Chaitin (grau / tamanho da live range) para selecção do candidato a spill (T2.4).

◆ algorithmParam

int AlgorithmConfig::algorithmParam = 0

Parâmetro numérico para as variantes spilling e splitting.

Para "spilling": número máximo de webs que podem ser enviadas para memória (spilled).

Para "splitting": número máximo de webs que podem ser divididas (split) antes de tentar a coloração.

Ignorado para "basic" e "free". O Parser define este campo a 0 quando não existe vírgula na linha do algoritmo.
