Detailed Description

Classe utilitária estática que implementa os algoritmos de alocação de registos.

O fluxo típico de utilização é:

InterferenceGraph g(ranges);
AlgorithmConfig cfg = Parser::parseRegisters("registers.txt");
auto [allocs, finalGraph, meta] = RegisterAllocator::allocate(g, cfg);
RegisterAllocator::printResult(finalGraph, allocs, std::cout, meta);

Aloca registos para todas as webs em graph e devolve o resultado.

Seleciona internamente o algoritmo indicado em config e retorna um triplo {alocações, grafofinal, metadados}:

  1. alocações: vetor de Allocation, um por web do grafo final. Para algoritmos sem modificação do grafo (basic, spilling, free), tem exatamente tantos elementos quantas as webs do grafo original. Para splitting, pode ter mais (as webs divididas originam novas entradas).
  2. grafo final: para basic, spilling e free é equivalente ao grafo original (cópia sem modificações); para splitting pode conter mais nós correspondentes às webs derivadas das divisões.
  3. metadados: string de texto em formato legível por máquina.
  • Vazia para basic e free.
  • Para spilling:
    # Spilling metadata [K=<param>]
    # Format: spilled_web: <webId> var=<name> degree=<d>
    spilled_count: <n>
    spilled_web: 2 var=i degree=5
  • Para splitting:
    # Splitting metadata [K=<param>]
    # Format: split: original_var=<name> split_at_line=<line> -> web<A> web<B>
    splits_count: <n>
    split: original_var=i split_at_line=9 -> web2 web3
Parameters
graphGrafo de interferência original.
configConfiguração com número de registos e variante do algoritmo.
Returns
Triplo {alocações, grafo final, metadados}.
Complexidade
O(W²) para basic/spilling/free; O(K × W² × L) para splitting. W = número de webs, L = linhas médias por web, K = parâmetro do algoritmo.

Coloração greedy de Kempe com limite opcional de spills.

Executa as fases de simplificação e coloração do algoritmo de Kempe:

Fase de simplificação:

  1. Percorre os nós ativos: se workDegree[n] < N, empilha n e decrementa o workDegree dos seus vizinhos ativos.
  2. Se o ciclo não remove nenhum nó E ainda há nós ativos, todos têm grau >= N: seleciona o candidato a spill (maior grau de trabalho) via selectSpillCandidate() e marca-o como spilled.
  3. Se spillCap >= 0 e o limite de spills é atingido antes de esvaziar active, define feasible = false e retorna antecipadamente sem executar a fase de coloração.

Fase de coloração: Desempilha um nó de cada vez e atribui-lhe a menor cor disponível (não usada por nenhum vizinho já colorido). Se nenhuma cor < N estiver disponível (grafo muito denso), o nó é marcado como spilled também nesta fase.

Log de spills: Se spillLog não for nullptr, cada web selecionada para spill na fase de simplificação tem o seu par (webId, grau) registado no vetor, por ordem de seleção. Usado pelo algoritmo spilling para construir os metadados de saída.

Parameters
graphGrafo de interferência.
NNúmero de registos físicos disponíveis (número de cores).
spillCapMáximo de spills permitidos; -1 = ilimitado.
feasibleSaída: true se a alocação respeita spillCap.
spillLogSaída opcional: lista de (webId, grau) das webs vertidas na simplificação, por ordem de seleção. nullptr por defeito — log não é preenchido.
Returns
Vetor de Allocation, um por web.
Complexidade
O(W²) onde W = número de webs (dominado pelo ciclo de simplificação).

Escreve o resultado da alocação num stream de saída.

Produz um ficheiro de saída com três secções:

Secção de webs — lista todas as webs do grafo final com os seus pontos de programa em ordem ascendente:

webs: 3
web0: 1+,2,3,4,5,6-
web1: 9+,10,11,12,13,14-,20+
web2: 7+,8,9,10-

Secção de registos — indica o número de registos efetivamente usados e, para cada web, o registo atribuído ou M:

registers: 2
r0: web0
r0: web1
r1: web2

Secção de metadados (opcional) — escrita apenas se metadata não estiver vazio; contém informação complementar sobre as operações de spill/split no formato descrito em allocate().

Parameters
graphGrafo de interferência final (pode ter mais webs que o original no caso de splitting).
allocationsResultado de allocate() — um Allocation por web.
outStream de saída (ficheiro ou std::cout).
metadataString de metadados complementares (valor por defeito "").
Complexidade
O(W × P log P), onde W = número de webs, P = pontos médios por web. O fator log P provém de toString(), que insere P pontos num std::map ordenado (O(log P) por inserção).

Seleciona o candidato a spill com maior grau de trabalho atual.

Itera sobre active e retorna o id do nó com o valor mais alto em workDegree. Ao remover o nó mais conectado, o grafo fica mais esparso, aumentando a probabilidade de a coloração subsequente ser viável com N cores.

Em caso de empate no grau, é selecionado o nó encontrado primeiro na ordem de iteração do std::set (menor id).

Parameters
workDegreeMapa id → grau de trabalho atual de cada nó ativo.
activeConjunto de ids dos nós ainda não removidos.
Returns
Id do nó com maior grau de trabalho.
Complexidade
O(|active|)

staticprivate

Divide uma web em duas metades no ponto mediano das suas linhas.

As linhas são ordenadas (já estão no std::set<int>) e divididas ao índice médio mid = size/2 (com mid >= 1 para evitar metades vazias em webs com uma única linha):

  • Linhas com índice < mid → primeira metade, variável com sufixo _s0.
  • Linhas com índice >= mid → segunda metade, variável com sufixo _s1.

Os pontos de programa de cada range original são distribuídos entre as duas metades de acordo com o conjunto de linhas a que pertencem. Se uma das metades ficar sem pontos, é preenchida com os pontos da outra (caso degenerado de web com uma única linha).

O parâmetro de saída splitLine recebe a primeira linha da segunda metade, que corresponde ao ponto de programa onde o compilador deve inserir as instruções de save/restore na fronteira da divisão.

Parameters
webWeb a dividir (deve ter pelo menos um ponto).
splitLineSaída: primeira linha da segunda metade (ponto de corte).
Returns
Par {primeiraMetade (_s0), segundaMetade (_s1)}.
Complexidade
O(P log P) onde P = total de pontos de programa da web.
