Aloca registos para todas as webs em graph.

Executa o algoritmo de alocação selecionado e devolve resultado + grafo final + string de metadados complementares.

Seleciona internamente o algoritmo indicado em config e devolve um triplo com:

  1. O vetor de alocações (uma por web do grafo final).
  2. O grafo de interferência final utilizado na coloração. Para basic, spilling e free é uma cópia do grafo original; para splitting pode ter mais nós (webs divididas).
  3. Uma string de metadados complementares em formato legível por máquina. Vazia para basic e free. Para spilling: lista das webs vertidas (id, variável, grau). Para splitting: lista das divisões efectuadas (variável, linha de corte, índices das webs derivadas).
Parameters
graphGrafo de interferência original.
configNúmero de registos e variante do algoritmo.
Returns
Triplo {alocações, grafo final, metadados}.
Complexidade
O(W²) para basic/spilling/free; O(K × W² × L) para splitting.

basic

Executa kempeWithSpillCap com spillCap=-1 (ilimitado). Apenas as webs que não conseguiram ser coloridas recebem "M" — as restantes mantêm o seu registo. Se alguma web foi vertida, emite aviso no stderr.

spilling

Itera cap de 0 até K (inclusive). Na primeira iteração viável, retorna. Os metadados indicam quantas webs foram vertidas, quais os seus ids, nomes de variável e grau de interferência no momento da seleção.

Formato dos metadados (spilling):

# Spilling metadata [K=<param>]
# Format: spilled_web: <webId> var=<name> degree=<d>
spilled_count: <n>
spilled_web: 2 var=i degree=5

splitting

Tenta primeiro sem modificações. Se falhar, entra no loop de splitting (até K iterações), dividindo sempre a web com maior grau. Os metadados indicam cada operação de divisão: qual a web original (por id e nome de variável), em que linha ocorreu o corte, e quais os ids das webs derivadas no grafo final.

Formato dos metadados (splitting):

# Splitting metadata [K=<param>]
# Format: split: original_var=<name> split_at_line=<line> -> web<A> web<B>
splits_count: <n>
split: original_var=i split_at_line=9 -> web2 web3

free (Chaitin)

Usa a razão grau/tamanho da live range para selecionar candidatos a spill.

Parameters
graphGrafo de interferência inicial.
configConfiguração com algoritmo e número de registos.
Returns
Tuplo {alocações, grafo final, string de metadados}.
Complexidade
O(W²) para basic/spilling/free; O(K × W² × L) para splitting.

Coloração greedy de Kempe com limite opcional de spills.

Simplificação e coloração greedy (Kempe) com limite opcional de spills.

Executa as fases de simplificação e coloração do algoritmo de Kempe. Na fase de simplificação, sempre que todos os nós restantes têm grau >= N, seleciona o candidato com maior grau de trabalho como spill. Se spillCap >= 0 e o número de spills atingir esse limite, define feasible = false e retorna antecipadamente (sem coloração). Se spillLog não for nulo, regista em cada spill o par (webId, grau).

Parameters
graphGrafo de interferência.
NNúmero de registos físicos disponíveis.
spillCapMáximo de spills; negativo = ilimitado.
feasibleSaída: true se a alocação respeita spillCap.
spillLogSaída opcional: lista de (webId, grau) das webs vertidas durante a simplificação, por ordem de seleção. Se nullptr, o log não é preenchido.
Complexidade
O(W²) onde W = número de webs.

Simplificação

  1. Percorre todos os nós ativos: se workDegree[n] < N, remove n para a pilha e decrementa o workDegree dos seus vizinhos ativos.
  2. Se o ciclo não removeu nenhum nó E ainda há nós ativos, seleciona o candidato a spill (selectSpillCandidate) e marca-o como spilled.
    • Se spillCap >= 0 e já atingimos spillCap spills, define feasible = false e retorna antecipadamente (sem coloração).

Coloração

Desempilha um nó de cada vez; recolhe as cores dos vizinhos já coloridos e atribui a menor cor não usada. Se nenhuma cor < N estiver disponível, o nó é marcado como spilled também nesta fase (grafos muito densos).

Log de spills

Cada vez que um nó é escolhido como candidato a spill na fase de simplificação, o seu id e grau são registados em spillLog para que o chamador possa incluir metadados no ficheiro de saída.

Parameters
graphGrafo de interferência.
NNúmero de registos físicos disponíveis.
spillCapLimite máximo de spills; -1 significa ilimitado.
feasibleSaída: true se a alocação respeita spillCap.
spillLogSaída opcional: lista de (webId, grau) das webs vertidas durante a simplificação (por ordem de seleção).
Returns
Vetor de Allocation, um por web.
Complexidade
O(W²) onde W = número de webs.

Escreve o resultado da alocação em out.

Imprime o resultado da alocação para um stream de saída.

Produz duas secções no formato exigido pelo enunciado:

  1. Lista de webs com os seus pontos de programa.
  2. Número de registos usados e atribuição de cada web a um registo ou M.

Se metadata não estiver vazia, é acrescentada uma terceira secção com os metadados complementares do algoritmo executado (spilling/splitting).

Parameters
graphGrafo de interferência final.
allocationsResultado de allocate().
outStream de saída.
metadataString de metadados (pode estar vazia; valor por defeito "").
Complexidade
O(W × P) onde W = número de webs, P = pontos médios por web.

Secção de webs

Lista todas as webs do grafo final (incluindo webs derivadas de splits) com os seus pontos de programa em ordem ascendente.

Secção de registos

Indica o número de registos efetivamente usados (pode ser inferior a config.numRegisters) e, para cada web, o registo atribuído ou "M" se foi vertida para memória.

Secção de metadados (opcional)

Se metadata não estiver vazia (algoritmos spilling e splitting), é escrita numa secção final do ficheiro no formato descrito em RegisterAllocator::allocate(). O formato é legível por máquina: linhas com palavras-chave específicas e separadores '=', '->', ' '.

Parameters
graphGrafo de interferência final.
allocationsResultado de allocate().
outStream de saída.
metadataString de metadados complementares (pode estar vazia).
Complexidade
O(W × P) onde W = número de webs, P = pontos médios por web.

Divide uma web ao ponto mediano das suas linhas de programa.

Divide uma web em dois no ponto mediano das linhas.

As linhas são ordenadas e divididas ao índice médio. A primeira metade recebe o sufixo _s0 no nome da variável, a segunda _s1. Em splitLine é devolvida a primeira linha da segunda metade, que corresponde ao ponto de programa onde o compilador deve inserir as instruções de save/restore necessárias à fronteira da divisão.

Parameters
webWeb a dividir; deve ter pelo menos um ponto de programa.
splitLineSaída: primeira linha da segunda metade (ponto de corte).
Returns
Par {primeira metade (_s0), segunda metade (_s1)}.
Complexidade
O(P log P) onde P = total de pontos da web.

A divisão é feita no índice médio do vetor ordenado de linhas.

  • Linhas com índice < média → primeira metade (sufixo _s0).
  • Linhas com índice >= média → segunda metade (sufixo _s1).

Casos importantes:

  • Se média == 0 (web com apenas uma linha), força mid=1.
  • Se uma das metades ficar vazia, copia os pontos da outra.
Parameters
webWeb a dividir.
splitLineSaída: primeira linha da segunda metade (ponto de corte).
Returns
Par {primeira metade, segunda metade} ambas com id=-1.
Complexidade
O(P log P) onde P = total de pontos da web.
