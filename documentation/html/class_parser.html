Detailed Description

Classe utilitária estática para leitura e parsing dos ficheiros de entrada.

Fornece dois métodos públicos estáticos para os dois ficheiros de entrada:

  • parseRanges() : lê o ficheiro de live ranges e devolve um vetor de LiveRange.
  • parseRegisters() : lê o ficheiro de configuração e devolve um AlgorithmConfig.

Os métodos privados trim(), isCommentOrEmpty() e parsePoint() são auxiliares partilhados pelas duas operações de parsing.

◆ parsePoint()

ProgramPoint Parser::parsePoint ( const std::string & token)
staticprivate

Faz o parse de um token individual de ponto de programa.

Percorre o token carácter a carácter:

  • Dígitos são acumulados na string digits.
  • '+' ativa a flag isStart.
  • '-' ativa a flag isEnd.
  • Outros caracteres são ignorados.

Se não for encontrado nenhum dígito, é lançada uma exceção. Tokens do tipo "7+-" (início e fim na mesma instrução, como em "i = i + 1") são corretamente tratados com ambas as flags ativas.

Parameters
tokenToken a parsear (e.g., "1+", "6-", "7+-", "4").
Returns
ProgramPoint construído a partir do token.
Exceptions
std::invalid_argumentSe o token não contiver nenhum dígito.
std::invalid_argument/ std::out_of_range Propagados por std::stoi se a conversão do número falhar.
Complexidade
O(n), onde n = comprimento do token.

std::vector< LiveRange > Parser::parseRanges ( const std::string & filename)
static

Lê e faz o parse do ficheiro de live ranges.

Para cada linha não-comentário e não-vazia:

  1. Divide pelo primeiro ':' para separar o nome da variável da lista de pontos de programa.
  2. Divide a lista de pontos por ',' e parseia cada token com parsePoint().
  3. Constrói um LiveRange e adiciona-o ao vetor de resultado.

Cada linha de dados origina exatamente um LiveRange, mesmo que a mesma variável apareça em múltiplas linhas. A fusão em webs é da responsabilidade de InterferenceGraph::buildWebs().

Parameters
filenameCaminho para o ficheiro de live ranges.
Returns
Vetor de LiveRange (um elemento por linha de dados).
Exceptions
std::runtime_errorSe o ficheiro não puder ser aberto.
std::invalid_argumentSe uma linha não contiver ':', se o nome da variável estiver vazio, ou se não houver nenhum ponto de programa válido.
Complexidade
O(L × P), onde L = número de linhas de dados, P = pontos por linha.

◆ parseRegisters()

AlgorithmConfig Parser::parseRegisters ( const std::string & filename)
static

Lê e faz o parse do ficheiro de configuração de registos.

Para cada linha não-comentário e não-vazia:

  1. Divide pelo primeiro ':' para obter chave e valor.
  2. Para a chave "registers": converte o valor para inteiro.
  3. Para a chave "algorithm": verifica se há vírgula no valor:
    • Sem vírgula → algoritmo sem parâmetro (basic ou free).
    • Com vírgula → algoritmo com parâmetro K (spilling ou splitting).
  4. Chaves desconhecidas são ignoradas silenciosamente.

Após ler todas as linhas, valida que numRegisters > 0.

Parameters
filenameCaminho para o ficheiro de configuração.
Returns
Estrutura AlgorithmConfig preenchida.
Exceptions
std::runtime_errorSe o ficheiro não puder ser aberto.
std::invalid_argumentSe uma linha não contiver ':', se o valor de registers estiver vazio ou for <= 0, se o parâmetro após a vírgula estiver vazio, ou se o algoritmo for desconhecido.
Complexidade
O(L), onde L = número de linhas do ficheiro.
