Aloca registos para todas as webs em graph.

Executa o algoritmo de alocação de registos selecionado na configuração.

Parameters
graphGrafo de interferência original.
configNúmero de registos e variante do algoritmo.
Returns
AllocResult com alocações e o grafo final usado.

basic

Chama kempeWithSpillCap com spillCap=-1 (ilimitado). Se não for possível colorir sem spills, os nós excedentes são marcados como spilled e é emitido um aviso no stderr.

spilling

Itera cap de 0 até K (inclusive) silenciosamente; emite uma única mensagem no stderr caso nenhuma iteração seja viável.

splitting

  1. Tenta primeiro sem modificações (cap=0, silencioso).
  2. Se falhar, entra no loop de splitting (até K iterações):
    • Seleciona a web com maior grau; se tiver só uma linha procura a próxima candidata divisível.
    • Divide-a com splitWeb() e reconstrói o grafo.
    • Tenta coloração do novo grafo com cap=0.
  3. Se após K splits ainda falhar, usa spills ilimitados (fallback) sobre o grafo mais refinado obtido.

free (Chaitin)

Usa kempeChaitin() que seleciona candidatos a spill pela razão grau_trabalho / tamanho_da_live_range.

Parameters
graphGrafo de interferência original.
configConfiguração com algoritmo e número de registos.
Returns
AllocResult com alocações e o grafo efetivamente colorido (pode diferir do original em splitting).
Complexidade
O(W²) para basic/spilling/free; O(K × W² × L) para splitting.

Coloração greedy de Kempe com limite opcional de spills.

Simplificação e coloração.

Parameters
graphGrafo de interferência.
NNúmero de registos físicos.
spillCapMáximo de spills; negativo = ilimitado.
feasibleDefinido como true se o resultado respeita spillCap.
silentSe true, suprime mensagens de aviso no stderr (usado nas iterações de sondagem do algoritmo spilling).

Simplificação

  1. Percorre todos os nós ativos: se workDegree[n] < N, remove n para a pilha e decrementa o workDegree dos seus vizinhos ativos.
  2. Se o ciclo não removeu nenhum nó E ainda há nós ativos, seleciona o candidato a spill (selectSpillCandidate) e marca-o como spilled.
    • Se spillCap >= 0 e já atingimos spillCap spills, define-se feasible=false e retorna-se antecipadamente.

Coloração

Desempilha um nó de cada vez. Para cada nó, recolhe as cores usadas pelos seus vizinhos já coloridos e atribui a menor cor não usada. Se a menor cor disponível for >= N, o nó é adicionalmente marcado como spilled (pode acontecer em grafos muito densos).

Parameters
graphGrafo de interferência.
NNúmero de registos físicos disponíveis.
spillCapLimite máximo de spills, se for negativo é ilimitado.
feasibleSaída: true se a alocação respeita spillCap.
Returns
Vetor de Allocation, um por web.
Complexidade
O(W²) onde W = número de webs.

Escreve o resultado da alocação em out.

Imprime o resultado da alocação para um stream de saída.

Parameters
resultResultado de allocate() (contém grafo e alocações).
outStream de saída.

Usa result.finalGraph para obter as webs efetivamente coloridas (que pode incluir webs novas criadas por splitting) e result.allocations para as atribuições de registo correspondentes.

Percorre as allocations duas vezes:

  1. Para calcular o número de registos efetivamente usados (maxReg+1, onde maxReg é o maior regIdx não-spilled).
  2. Para emitir as linhas de atribuição ("rN: webM" ou "M: webM").
Parameters

outStream de saída.
Complexidade
O(W × P) onde W = número de webs, P = pontos médios por web.

Seleciona o candidato a spill com maior grau.

Parameters
workDegreeMapa id -> grau de trabalho atual de cada nó ativo.
activeConjunto de ids dos nós ainda não removidos do grafo.

Percorre todos os nós ativos e regista o nó com o maior grau de trabalho. Em caso de empate, o nó com id menor é escolhido porque o set está ordenado.

Ao remover o nó mais conectado, o grafo tende a tornar-se mais esparso e a coloração seguinte é mais provável de ser viável.

Divide uma web no ponto mediano das linhas de programa.

Divide uma web em dois no ponto mediano das linhas.

Parameters
webWeb a dividir; deve ter pelo menos um ponto de programa.

A divisão é feita no índice médio do vetor ordenado de linhas.

Linhas com índice < média, corresponde à primeira metade. Linhas com índice >= média, corresponde à segunda metade.

Casos importantes:

  • Se média == 0 (web com apenas uma linha), força mid=1, colocando a única linha na primeira metade.
  • Se uma das metades ficar vazia após a distribuição, copia os pontos da outra (evita um LiveRange sem pontos, que seria inválido).
Parameters
webWeb a dividir.
Returns
Par {primeira metade, segunda metade} ambas com id=-1.
Complexidade
O(P log P) onde P = total de pontos da web.
